1. 프로젝트 개요
--------------------------------------------------------------------------------

  본 프로젝트는 AI를 활용하여 기업의 브랜드 스토리를 추출하고,
  이를 바탕으로 홍보 웹페이지를 구성하는 전 과정을 학습하기 위해
  제작되었습니다.

  대상 기업: 진삼가(주식회사 진삼) — 9증9포 명품홍삼 브랜드
  기업 동의: 진삼가의 동의를 받아 실제 브랜드 정보를 활용
  목적: 학습 및 포트폴리오 (상거래 기능 미포함)

  배포 URL: https://pagehongsam.vercel.app
  GitHub: https://github.com/ORN-Consulting/page_hongsam


2. 제작 과정 (워크플로우)
--------------------------------------------------------------------------------

  전체 과정은 다음 8단계로 진행되었습니다.

  [1단계] 기존 홈페이지 분석 및 콘텐츠 수집
    - jinsamga.co.kr 웹사이트를 Firecrawl로 스크래핑
    - 제품 정보(11개 제품), 이미지, 가격, 성분 데이터 수집
    - 브랜드 메시지, 회사 정보, 고객센터 정보 추출

  [2단계] 브랜드 스토리 정제
    - 수집된 원시 데이터를 브랜드 스토리 문서로 정제
    - 6개 챕터 구조 설계 (사람/뿌리/기술/정직/거점/철학)
    - 브랜드 전문가 관점의 진정성 검토 및 보완

  [3단계] 디자인 시스템 수립
    - Google Stitch MCP를 활용한 초기 디자인 생성
    - DESIGN.md로 디자인 시스템 문서화
      (색상 팔레트, 타이포그래피, 엘리베이션, 컴포넌트 규칙)
    - Cinematic Landing Page Builder 기준 적용
      (noise overlay, magnetic buttons, GSAP animations)

  [4단계] 폰트 시스템 설계
    - 한글 웹폰트 가이드 작성 (jinsamga_web_font_guide.md)
    - Gowun Batang(타이틀 세리프) + Gowun Dodum(본문 산세리프) 조합 선정
    - 역할 분담, 줄간격, 자간 등 상세 가이드 수립

  [5단계] 페이지별 구현
    - 메인 랜딩 페이지 (main-landing.html)
    - 브랜드 스토리 페이지 (product-story.html)
    - 제품 목록 페이지 (products.html)
    - 제품 상세 페이지 (product-jin.html)

  [6단계] 제품 페이지 진단 및 재설계
    - 기존 jinsamga.co.kr 제품 페이지의 문제점 진단 문서 작성
    - 고객 관점 정보 설계로 전환하는 개선안 수립
    - 10섹션 제품 상세 페이지 구성 제안서 작성 및 구현

  [7단계] 품질 검수 및 시네마틱 업그레이드
    - Cinematic Landing Page Builder 기준으로 전 페이지 검토
    - 7개 카테고리별 점수 산출 후 전면 개선
    - 프리뷰 서버를 통한 실시간 확인 및 반복 수정

  [8단계] 배포
    - GitHub 리포지토리 생성 및 push
    - Vercel을 통한 프로덕션 배포

4. 방법론: AI 기반 브랜드 웹페이지 구축 프레임워크
--------------------------------------------------------------------------------

  본 프로젝트에서 검증한 방법론은 다음과 같습니다.

  4-1. 콘텐츠 퍼스트 (Content-First)
    디자인보다 콘텐츠를 먼저 확보합니다.
    - 기존 웹사이트에서 텍스트, 이미지, 제품 데이터를 자동 수집
    - 수집된 원시 데이터를 브랜드 스토리 문서로 정제
    - 정제된 콘텐츠를 기반으로 페이지 구조를 설계

  4-2. 디자인 시스템 드리븐 (Design System-Driven)
    개별 페이지가 아닌 시스템을 먼저 만듭니다.
    - DESIGN.md로 색상, 타이포그래피, 컴포넌트 규칙을 문서화
    - 모든 페이지가 동일한 디자인 토큰을 공유
    - 폰트 가이드 문서로 한글 타이포그래피 일관성 확보

  4-3. 프롬프트 체이닝 (Prompt Chaining)
    하나의 거대한 프롬프트가 아닌, 단계적 프롬프트 체인을 사용합니다.
    - 1차: Stitch로 초기 디자인 생성
    - 2차: 실제 콘텐츠로 교체
    - 3차: 디자인 시스템 적용
    - 4차: Cinematic 기준 검토 및 업그레이드
    - 5차: 사용자 피드백 기반 세부 조정

  4-4. 진단 기반 재설계 (Diagnosis-Based Redesign)
    기존 페이지의 문제를 체계적으로 진단한 후 개선합니다.
    - 구조/정보설계, 제품 상세, UX/디자인, 전환율 4개 카테고리로 진단
    - 심각도별 우선순위 도출
    - 고객 심리 흐름에 맞춘 섹션 순서 설계

  4-5. 실시간 검증 루프 (Live Verification Loop)
    코드 작성과 시각적 검증을 동시에 수행합니다.
    - 로컬 프리뷰 서버 상시 가동
    - 코드 수정 즉시 스크린샷으로 결과 확인
    - 사용자 피드백을 즉각 반영하는 반복 사이클


5. 프로젝트 특징
--------------------------------------------------------------------------------

  5-1. 시네마틱 품질 기준 적용
    일반적인 템플릿 수준이 아닌, 프리미엄 에디토리얼 웹사이트 수준을
    목표로 했습니다.
    - SVG feTurbulence 노이즈 오버레이 (디지털 평면감 제거)
    - Pill 형태 플로팅 네비게이션 (스크롤 시 투명/블러 모핑)
    - Magnetic 버튼 (scale 1.03 + sliding background layer)
    - GSAP ScrollTrigger 기반 fade-up 애니메이션
    - rounded-[2rem]~[3rem] 컨테이너 시스템
    - 다크 푸터 (rounded-t-[4rem] + System Operational 인디케이터)

  5-2. 한글 타이포그래피 최적화
    한글 가독성을 최우선으로 고려한 폰트 시스템입니다.
    - Gowun Batang: 전통/정성/자연의 온기를 전달하는 세리프
    - Gowun Dodum: 부드러운 가독성의 산세리프
    - 줄간격 1.4~1.8, 자간 -0.02em~0.05em 세밀 조정
    - SIL Open Font License (무료, 상업용 가능)

  5-3. 고객 관점 정보 설계
    "제품 나열"이 아닌 "구매 결정을 돕는" 구조입니다.
    - 목적별 탭 필터링 (나를 위한/선물용/자녀/간편 휴대)
    - 제품 카드에 진세노사이드 배수, 핵심 차별점 표시
    - 7개 제품 비교 테이블 (이미지 포함)
    - 10섹션 제품 상세 페이지 (관심→이해→신뢰→확신→구매 흐름)

  5-4. 브랜드 스토리텔링 구조
    6개 챕터로 구성된 브랜드 내러티브입니다.
    - Chapter 1. 사람 — 대표 김명범의 홍삼 외길 스토리
    - Chapter 2. 뿌리 — 풍기 6년근, 1,500년 인삼 역사
    - Chapter 3. 기술 — 9증9포, 세계 최초 전자동 시스템
    - Chapter 4. 정직 — 합성첨가물 제로, 성분표가 곧 자부심
    - Chapter 5. 거점 — 부산에서 세계로, 글로벌 네트워크
    - Chapter 6. 철학 — 신념/태도/행동/헌신 네 가지 가치

  5-5. MCP 기반 AI 에이전트 활용
    단순 코드 생성이 아닌, 여러 AI 도구를 연결하여 사용했습니다.
    - Stitch MCP: 디자인 생성 및 관리
    - Firecrawl MCP: 웹 콘텐츠 자동 수집
    - Claude Preview: 실시간 브라우저 프리뷰
    - Notion MCP, Slack MCP 등 확장 가능한 도구 생태계

7. 학습 포인트 요약
--------------------------------------------------------------------------------

  [1] AI는 "코드를 짜는 도구"가 아니라 "브랜드를 이해하고
      고객 관점으로 재구성하는 파트너"로 활용할 수 있다.

  [2] 디자인 시스템을 먼저 수립하면, AI가 생성하는 모든 페이지의
      일관성이 자동으로 확보된다.

  [3] 기존 웹사이트의 문제를 체계적으로 진단하는 것이
      단순히 "예쁘게 다시 만드는 것"보다 훨씬 가치 있다.

  [4] 프롬프트 한 번으로 완성되는 것이 아니라,
      단계적 체이닝과 반복 피드백이 품질을 결정한다.

  [5] 한글 타이포그래피는 영문과 다른 고유한 고려사항이 있으며,
      이를 문서화하면 AI도 일관된 품질을 유지할 수 있다.
